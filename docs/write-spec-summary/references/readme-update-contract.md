# README Dashboard Update Contract — Spec Summary Layer

After producing a spec summary, update the subsystem's `README.md` dashboard.

## Location

The README lives at the same directory level as the summary: `docs/{subsystem}/README.md`.

## If README Does Not Exist

The spec skill should have created it. If not, create it following the template in the spec layer's readme-update-contract (create a minimal version with the Phase Map and Status Matrix).

## Update Procedure

1. **Status Matrix** — Set the Spec Summary cell for this phase to `[Done]({filename})`
2. **Last updated** — Set to today's date

The spec summary does not introduce cross-phase dependencies or carry-forward contracts (those come from the spec itself), so no other sections need updating.

## Status Values

- `—` = not started
- `[Done]({filename})` = completed, with link to document
