# registry

Human-readable discovery tables for all artifacts in the repo. Check here before creating a new skill, agent, protocol, tool, or script — an existing one may already cover what you need.

## Files

| File | Description |
|------|-------------|
| [AGENTS_REGISTRY.md](AGENTS_REGISTRY.md) | All agent definitions — internal recipes dispatched by skills |
| [PROTOCOL_REGISTRY.md](PROTOCOL_REGISTRY.md) | All protocols — behavioral contracts injected into agents |
| [SKILL_REGISTRY.md](SKILL_REGISTRY.md) | All skills — user-invocable recipes with domain, pipeline stage, and status |
| [TOOL_REGISTRY.md](TOOL_REGISTRY.md) | All tools — standalone utilities with domain, action, and status |
| [SCRIPT_REGISTRY.md](SCRIPT_REGISTRY.md) | All scripts — repo management scripts with audience, action, and scope |
| [PATHWAY_REGISTRY.md](PATHWAY_REGISTRY.md) | All pathways — named synapse bundles with harness and description |
