# docs

Document authoring, review, and maintenance — section writers, section reviewers, and doc-level coherence reviewers.

## Agents

| Agent | Role | Description |
|-------|------|-------------|
| [docs-spec-section-writer](docs-spec-section-writer.md) | writer | Writes one spec section from a planner brief — requirement format, acceptance criteria, traceability |
| [docs-spec-section-reviewer](docs-spec-section-reviewer.md) | reviewer | Three-way evaluator — brief alignment, section quality, deviation justification |
| [docs-spec-coherence-reviewer](docs-spec-coherence-reviewer.md) | reviewer | Doc-level coherence review — alignment, flow, cross-references, traceability |
| [docs-epic-writer](docs-epic-writer.md) | writer | Writes epic.md cross-cutting deltas + vocab.md (declarative-only) |
| [docs-fr-decomposer](docs-fr-decomposer.md) | writer | Decomposes brainstorm memo into FR roster + per-FR atomic tmp context files |
| [docs-story-writer](docs-story-writer.md) | writer | Vertical-slice writer — story.md + design.md + impl.md + test.md for one FR |
| [docs-eng-guide-writer](docs-eng-guide-writer.md) | writer | Regenerates engineering_guide.md from touches: union (stable skeleton) |
| [docs-eng-guide-reviewer](docs-eng-guide-reviewer.md) | reviewer | Drift-checks regenerated eng-guide vs prior — emits clean/reconcile/escalate |
