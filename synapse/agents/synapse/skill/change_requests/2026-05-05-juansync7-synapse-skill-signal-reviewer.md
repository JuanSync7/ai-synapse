# Decision Memo — synapse-skill-signal-reviewer

> Artifact type: agent | Memo type: creation | Design doc: `.brainstorms/2026-05-05-synapse-skill-signal-reviewer/notepad.md`

---

## What I want

A thin orchestrator agent that runs a graded, multi-dimensional review of skill authoring artifacts (SKILL.md + companion files) before eval generation. The orchestrator dispatches three specialized sub-agents in parallel — anatomy-reviewer, design-judge, companion-auditor — collects their verdicts, and returns a single unified APPROVE / REVISE / ESCALATE signal to the dispatcher.

Scope: all skill files in a given skill directory (SKILL.md + references/ + templates/). EVAL.md is explicitly out of scope — it belongs to a separate lifecycle stage, generated by write-synapse-eval.

---

## Why Claude needs it

Without this agent, skill review at creation time is ad-hoc. The existing `/improve-skill` skill operates reactively on outputs after-the-fact; there is no proactive structural gate that fires before eval generation. The result is:

- Skills reaching the eval generation stage with anatomy defects (missing frontmatter fields, descriptions that summarize the workflow rather than act as a routing contract, absent Wrong-Tool Detection).
- Design quality issues (procedure instead of policy, no failure-mode tracing, bloated instructions) that go undetected until a costly post-eval improvement loop.
- Companion files (references/, templates/) that are orphaned, undocumented in SKILL.md, or duplicate the body — silently degrading progressive disclosure.

The signal-reviewer closes this gap by inserting a graded, structured gate at the `[R]` phase of synapse-creator's flow-skill, before `[EVAL]` handoff.

---

## Injection shape

- **Workflow:** Thin orchestrator — no multi-phase flow of its own. Receives skill directory path, dispatches 3 sub-agents in parallel, aggregates verdicts, returns unified output.
- **Policy:** Aggregation rule (APPROVE / REVISE / ESCALATE thresholds), fix-cycle ownership (lives in dispatcher, not orchestrator), output labeling convention.

## Pipeline integration (VERBATIM)

<!-- VERBATIM -->
**Pipeline integration (synapse-creator flow-skill `[R]` phase):**
```
Load: agents/skill/synapse-skill-signal-reviewer.md
Do:
  1. Dispatch synapse-skill-signal-reviewer (model: sonnet) with skill directory path.
  2. If verdict = REVISE: fix issues, re-dispatch. Two cycles max.
  3. If verdict = ESCALATE after 2 cycles: surface to user.
Exit: APPROVE → [EVAL]
```

---

## Aggregation rules

- **APPROVE:** anatomy 100% pass AND design avg ≥ 3.5 AND no design dimension < 2 AND companion checks ≥ 90% pass.
- **REVISE:** any anatomy fail OR design avg < 3.5 OR any dimension < 2 OR companion pass rate < 90%.
- **ESCALATE:** REVISE verdict on second cycle — signals a likely design issue, not a wording fix.

Fix loop (2 cycles max) lives in the **dispatcher** (synapse-creator flow-skill `[R]` phase), not inside this orchestrator. The orchestrator is stateless — it reviews, aggregates, and returns. It does not iterate.

---

## Sub-agents dispatched

Three sub-agents are dispatched in parallel. Each is its own creation (separate memos):

| Sub-agent | Role | Spec source loaded at runtime |
|-----------|------|-------------------------------|
| `synapse-skill-anatomy-reviewer` | Binary gate on SKILL.md structural anatomy | CLAUDE.md skill anatomy section + synapse-creator flow-skill drafting rules |
| `synapse-skill-design-judge` | Graded 1-5 per design-quality dimension | `synapse/skills/skill/skill-creator/references/skill-design-principles.md` |
| `synapse-skill-companion-auditor` | Audit references/ + templates/ progressive disclosure | progressive-disclosure rules in skill-design-principles.md |

The orchestrator collects their raw outputs, labels findings by source (`[anatomy]`, `[design]`, `[companion]`), and applies the aggregation rule to produce the unified verdict.

## Edge cases considered

| Edge case | Handling |
|-----------|----------|
| Skill has no references/ or templates/ | companion-auditor skips companion-file checks (no orphan check possible); pass rate defaults to 100% for that section |
| `status: draft` in skill frontmatter | Reviewer is stateless on lifecycle — reviews regardless of status field |
| Wrong-Tool Detection absent but skill is user-invocable | anatomy-reviewer flags FAIL on check 4; design-judge may also penalize on progressive-disclosure dimension |
| Design dimension scores < 2 | REVISE verdict regardless of anatomy or companion results; orchestrator labels the specific dimension(s) in findings |
| Second REVISE cycle | Orchestrator returns ESCALATE; dispatcher surfaces to user — likely a structural design issue that requires author judgment, not a wording fix |
| Missing spec source file (e.g., skill-design-principles.md not found) | Each sub-agent loud-fails with clear error per failure-reporting protocol; orchestrator surfaces the sub-agent failure without proceeding to aggregation |
